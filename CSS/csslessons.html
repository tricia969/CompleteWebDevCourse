Learning CSS

CSS stands for Cascading Style Sheets. The Cascading part of the name refers to the fact that multiple styles can be applied to a single document with only the last style for any element being the one that is displayed.

Inline Style

Quickly change the look of a single element by adding styling to the tag. This is not recommended for any styling that would appear in multiple instances as it would require coding each instance and then editing each instance separately if changes were desired.

Code: <p style="color:blue; font-size:150%;">

Internal Style

Internal styling happens in the header of a document. Unless specifically denoted it will affect all elements of the declared type (body, paragraph, div). It will not however overwrite inline styling because of the order of style application (external, internal, inline).

Classes & IDs

Classes and IDs allow you to alter styles across elements and also to designate some instances of an element as different. Classes may be used multiple times in a single document, while styling rules dictate that an ID only appear once in each document.

Class names are preceded by a . when declaring them in the style section. A style can define new properties for an element and/or overwrite existing properties for the element it is applied to. Any styles not overwritten by the class will follow previously declared style rules or the default styling if no specific style has been declared.

An ID can be used to style a single element in a document. ID are useful when applying javascript to a page. ID names are preceded by a # when declaring them in the style section. Like classes IDs can define new properties and/or overwrite previous declarations.

You can use both an ID and a class to define the properties of an element. Stacking styles can get tricky when the same property is defined multiple times. Things like the order the styles are applied, the importance of the style, and whether the style is defined as a subtype of the original style all play a role in the final look.

You may define as many characteristics as you want in a class, ID, or element style definition. This paragraph has a special font-family, color, and size all defined by the class curlygreen.

Code: 
        <style type="text/css">
            p {
                color:red;
            }
            
            .tiny {
                font-size:75%;
            }
            
            .purple {
                color:purple;
            }
            
            @font-face {
                font-family: Vladimir;
                src: url(Fonts/vladimir-webfont.woff);
            }
            
            #curly {
                font-family:Vladimir;
            }
            
            #big {
                font-size:125%;
            }
            
            .blue {
                color:blue;
            }
            
            .curlygreen {
                font-family:Vladimir;
                font-size: 135%;
                color:green;
            }
        </style>

divs

The div tag provides a logical way to divide up your page and keep sections logically ordered and defined. A div appears as a rectangle surrounding the content. You can control all the properties of a div the same way you do any other element.

Apply styles to the base element div carefully, because every div will adopt that styling. Using an ID for a sectional div or a class for a lower level div will allow easier customization and prevent having to overwrite too many style rules.

In the bluebox div I defined the color of the text as white, but the paragraph tag inside the div overwrites the text color to red. In the pinkbox div I declared the text color blue and didn't use a paragraph tag so the blue text shows. In this yellowbox div I added a style definition that changes the color of paragraph text inside yellowboxes to purple allowing the desired text color to show. The yellowbox example is considered stylistically correct.

Color

In the past browsers had limited color capability and differed greatly in their color displays, this is no longer the case. As we've seen a few basic colors have defined names that are understood by browsers like red, blue, green, purple, etc. If you want a specific color or a color not name defined you can use hexidecimal color codes to define very specific color. Programs like Photoshop and Paintshop will provide you with a hex code when you use the color palette or you can visit a website like html color codes or w3c html color picker to pick a color and obtain its hex code. Don't forget to use the # symbol in front of your six character code.

Need help picking colors that work well together try a website or app that develops color palettes with hex codes for you. Some tools to try include:

  • Colour Lovers - includes user shared palettes)
  • Adobe Color CC - develops palettes based on your choice of color rule (formerly Adobe Kuler)
  • Collor - easy to use
  • Contrast-A - good for ensuring contrast and readability
  • Color Blender - offers suggestions but allows customization
  • Coolors - web & ios app

positioning

The position property can have a value of static, fixed, relative or absolute. Static is the default value and static elements sit where they fall in the normal flow of the page.

Fixed elements are removed from the normal flow of the page. A fixed element will be positioned the specified distance from one corner of the page (use left, top, right, and/or bottom) and won't move even when the screen scrolls.

Relative is positioned relative to its normal page position and its normal space is preserved. Relatively positioned items can be overlapped and the order of the overlapping elements can be determined by defining the z-index.

The z-index can be positive or negative and higher z-index values always appear in front of lower z-index values.

Changing the z-index from its default (shown above) will change the order of the layers regardless of their original flow position.

Relative is positioned relative to its normal page position and its normal space is preserved. Relatively positioned items can be overlapped and the order of the overlapping elements can be determined by defining the z-index.

Absolute elements are positioned relative to the first non-static parent element or if none exist the html tag. Absolutely positioned elements are removed from the page flow and are ignored by the other page elements. Like relative elements absolute elements can overlap and their order defined by the z-index.

margin

You can change the position of an element on the page by adjusting its margins. Elements repositioned by margin adjustments do not exit the flow of the document and generally do not overlap.

All of the margins can be defined equally by using margin:#; the number will be used to create margins on the top, right, bottom, and left sides.

If you want to declare different margin sizes you can use margin: top# right# bottom# left#; to define each value.

If you want to declare just one margin you can use margin-top, margin-right, margin-bottom, or margin-left.

The values used for margin can be a length using px, pt, cm, etc. to define the exact size of the margin. Margins can also be defined as a percent of the containing element. Margin values can be positive or negative. Negative values can cause element overlap.

padding

Padding acts much like margins but the space is created inside the element rather than outside. Unlike margins the padding is added to the amount of space an element takes up. Like margins padding can be length or %. padding:#; will create the same amount of padding all around.

Using padding: top# right# bottom# left#; will create different amounts of padding as specified.

Like margin, padding can also be defined for a single side at a time using padding-top, padding-right, padding-bottom, or padding-left.

border

Borders like padding add to the size of your element, a 100x100 div with a 20px border all around takes up 140x140 pixels not just the 100px declared. Using border you can declare the size, color and style of your border.

You can also define each part of the border separately using border-style, border-width, and border-color. You can opt to declare a set value for all four sides or define each side as a different value.

You can also define the border-radius to create nice rounded corners on your divs. If you set your border-radius to half the width of your div on all four sides of a square div you'll get a circle, rectangular div will give you an oval.

Styling Text

Use font-weight to define the thinness/thickness of your font. The values for font-weight are bold, bolder, lighter, 100, 200, 300, 400, 500, 600, 700, 800, or 900. The paragraph is set to bold but each additional option is spanned to its own value.

The color attribute changes the text color. As mentioned before you can use hex codes and color names to define colors. You can also use rgb, which stands for Red Green Blue, like this color: rgb(0,0,255);.

The font-size attribute changes the size of the text. Font size can be defined by pixels (px), percentage of the surrounding text (%), or em where 1em is equal to the current size font.

The font-style attribute sets text to normal, italic, and oblique. Oblique is similar to italic but not as widely supported.

The text-decoration attribute has the values overline, underline, and line-through.

The text-transform attribute has the values uppercase (set all letters to uppercase), lowercase (sets all letters to lowercase), or capitalize (sets the first letter of each word to uppercase).

The text-indent attribute indents the first line of text the designated number of pixels. This paragraph has an 80px indentation.

Aligning Text

(text-align:left;) Text in a document defaults to left alignment, you can declare it if you like or if you need to override inherited rules. Left aligned text fills from left to right and partial lines have blank space to the right.

(text-align:right;) A paragraph without a containing element will align to the page itself. Right aligned text is flush with the right margin and partial lines will have blank space to the left.

(text-align:center;) If a paragraph has a parent element with a declared alignment and no alignment declaration of its own it will inherit the parent's alignment. Center aligned text will fill from the middle of and partial lines will have blank space to the right and left.

(text-align:justify;) Text aligned to justify will stretch each full line to cover the width of the page or parent element. Lines fill from left to right and partial lines have blank space to the right. Text aligned to justify will stretch each full line to cover the width of the page or parent element. Lines fill from left to right and partial lines have blank space to the right.

Styling Links

default styling

A unvisited link will will be blue, a visited link will be purple, and a hovered link will be blue or green depending on its visited status. Links are underlined by default.

How to Style

Links can be styled by defining the attributes in the <a> tag. Using a:link (normal unvisited link), a:visited, a:hover (link with cursor over it), and a:active (moment link is clicked) you can easily define the appearance of every link in a document during ever state.

Google

Google SE

Code: 
        .linkmod a:link {
            color:Orange;
            font-weight:bold;
            text-decoration: none;
        }
        
        .linkmod a:visited {
            color:OliveDrab;
            text-decoration: none;
        }
        
        .linkmod a:hover {
            color:DarkCyan;
            font-size: 250%;
            text-decoration:none;
        }
        
        linkmod a:active {
            color:Indigo;
            font-size: 500%;
        }

External Style

Because even a fairly simple document's styling can quickly cover multiple screens which make documents harder to read and causes the styling to be downloaded with each page accessed styling is often stored in a seperate style document. The style document can then be included in the document's header. A document may use multiple styling documents, styles will be applied in the order the document are called.

An external style sheet simply contains elements and attribute definitions and is of type .css. A link tag is included in the header that attaches the stylesheet to the document.

Code: 
        <link rel="stylesheet" type="text/css" href="style.css" />
